# Pulse Project Status — 2026-02-24

## 1. What This Project Is

Pulse is a personalized social discovery app for Denver, CO. It aggregates events and places from multiple sources, enriches them with AI, and scores/ranks them against each user's preferences — vibe, companion type, budget, schedule, dog-friendliness, sober-friendliness, and more.

**Target audience:** Denver residents and newcomers looking for things to do — date nights, group outings, solo adventures, family activities.

**Business context:** Solo founder (Quest Taylor), live in production with real users, actively seeking a co-founder. Production URL: https://pulse-three-eta.vercel.app

**Core stack:**
- Next.js 14 (App Router) / TypeScript
- PostgreSQL on Neon (serverless) via Prisma 5
- NextAuth.js 4 (email/password credentials)
- OpenAI GPT-4o-mini (event & place enrichment)
- Cheerio + Ticketmaster API + Eventbrite API (scraping)
- Tailwind CSS 3 / Lucide React
- Vercel (hosting, cron jobs, blob storage)

## 4. Key Decisions Made

### Place vs Item dual-model (critical context)
The database has **two separate models** for places:
- **`Place`** — canonical venue data from Google Places API (lat/lng, ratings, hours, enrichment). Used by `/places` UI and detail pages.
- **`Item`** — legacy unified model (type: EVENT | PLACE). `UserItemStatus` (Want/Done/Pass) and `UserItemRating` link to Item, not Place.

The **bridge pattern** avoids a schema migration: `setPlaceStatus(placeId)` finds-or-creates a matching Item record (by name + address), then upserts status on that Item. A proper fix would add `placeId` to `UserItemStatus` or create a dedicated `UserPlaceStatus` table.

### Visit Denver rejected
Visit Denver uses SimpleView CMS which renders events via JavaScript. Can't scrape with Cheerio, and Puppeteer won't run on Vercel serverless. Ticketmaster + Eventbrite APIs were chosen instead for structured JSON, good Denver coverage, and serverless compatibility.

### Chunked cron for places refresh
12 Google Places categories can't all refresh within Vercel's 60s function limit. Split into 3 chunks of 4 categories, staggered 10 minutes apart on Sundays.

### Electric Blue color system
Primary color standardized to `#2563EB` (Tailwind blue-600). Applied globally across all components in the UI standardization pass.

### Conditional scraper registration
Ticketmaster and Eventbrite scrapers are only added to the scraper array if their env vars are set. No errors or warnings if keys are missing — they just don't run.

### Classifier: venue-first, keywords-second
Event classification checks a 65+ entry venue map first (e.g., Red Rocks -> LIVE_MUSIC, Denver Art Museum -> ART), then falls back to weighted keyword matching with word-boundary regex. Venue map takes priority, which occasionally produces wrong results when a venue hosts off-category events.
